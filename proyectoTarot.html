Diario de Tarot

Aplicación web de ocio

Estado: En desarrollo

Aplicación personal para registrar lecturas de tarot y reflexiones en un diario digital.
Este proyecto lo inicié como un regalo para una amiga, con el objetivo de crear una herramienta sencilla y visual, que en el futuro, pueda evolucionar hacia un sistema con almacenamiento persistente y versión móvil.

¿Qué he aprendido?

  • Ha sido el primer proyecto en el que he usado JavaScript
  • Manipulación del DOM

Especificaciones principales:

Añadir lectura (en desarrollo)

  • El usuario puede seleccionar cartas de un menú dinámico generado con JavaScript
  • Se incluye un calendario para elegir la fecha de la lectura.
  • El formulario permite introducir un texto descriptivo de la lectura.
  • Objetivo: almacenar conjuntamente texto, fecha y cartas seleccionadas para futuras consultas.

Ver Diario (en desarrollo)

  • Sección destinada a mostrar las entradas previamente añadidas.
  • Objetivo: recuperar y listar las lecturas guardadas, organizadas por fecha.
  • Próximas mejoras: filtros de búsqueda y posibilidad de editar/eliminar entradas.

Carta del día (en desarrollo)

  • El usuario puede pulsar un botón para que aparezca una carta seleccionada al azar.
  • Las cartas se gestionan mediante un array en JavaScript.
  • Funcionalidad pensada como recurso rápido para obtener una tirada simple.

Planes de futuro

Este proyecto está todavía en desarrollo, lo que me permite seguir explorando nuevas herramientas y plantear mejoras que integren diferentes lenguajes y enfoques.

Persistencia de datos

Integrar almacenamiento en base de datos o mediante una API REST en Java, para que las entradas no se pierdan al cerrar la aplicación.

Gestión del diario

Añadir funcionalidades de edición y eliminación de entradas, así como filtros por fecha o carta.

En cuanto a los filtros, mi idea es implementar un endpoint que devuelva las cartas mediante peticiones GET una vez integrada la persistencia de datos, y posteriormente aplicar un filtrado dinámico en el cliente utilizando JavaScript

Diseño responsive

Adaptar la interfaz para que se use cómodamente en dispositivos móviles y tablets utilizando media queries.

Versión móvil

Explorar el desarrollo de una Progressive Web App (PWA) o aplicación Android sencilla que consuma los datos desde la API.
